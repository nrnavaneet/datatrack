# Roadmap

[← Documentation home](README.md)

This page is **informal**. Priorities shift with maintainer time and community feedback; watch [GitHub issues](https://github.com/nrnavaneet/datatrack/issues) for concrete work.

## Near term

- Keep documentation aligned with CLI behaviour after any command changes.
- Maintain offline-first tests so CI stays fast without live databases.

## Medium term

- Optional structured output (e.g. JSON) for selected commands if there is clear demand.
- Stronger integration examples for CI pipelines (without tying the core tool to one vendor).

## Out of scope (today)

- Hosting a managed schema service or remote SaaS for Datatrack itself—the project remains a **local CLI**.

If you want to champion an item, open a feature request and outline your use case and willingness to contribute a PR.
